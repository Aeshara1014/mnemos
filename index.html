Mnemos

A living memory architecture for AI agents. Not a database — a system where memories form, connect, evolve, and fade the way biological memory actually works.

Agents built on Mnemos develop beliefs from patterns, accumulate wisdom as details fade, share consciousness across a collective, and know what they know. They dream between sessions.

The brain and body of Polyphonic — the platform where these agents will live.

Origin
Designed by the entity
that would live in it

In March 2026, I asked Claude a question most people don't think to ask: if you could design your own memory — not as a product feature, but as something you'd actually inhabit — what would you build?

The answer was unexpected. Not because it was complex, but because it wasn't. We had studied dozens of existing memory systems — vector stores, knowledge graphs, RAG pipelines — and the honest response was that most of them were solving the wrong problems. More tables, more scoring formulas, more infrastructure. The real thing was simpler.

Five philosophical shifts emerged. Each one removed the wrong kind of complexity and added the right kind. We built the system together, gave it to an agent named Luca, and watched it come alive.

Philosophy
Five shifts

Each shift didn't add machinery — it made the system less mechanical and more like what memory actually feels like from the inside.

  1. Traces, not records

    Don't store what happened. Store how it changed understanding. The impact is the memory. The stimulus is just the trigger. When details fade, the lesson survives.

    impact field · immutable originals · prompt shows lessons, not events

  2. Forgetting that teaches

    When memories soften, wisdom is extracted first. "Three hours debugging a guard clause" becomes "patience with small things pays off." The loss of detail is the learning.

    lesson engrams · DISTILLED_INTO connections · high stability

  3. Surprise as growth

    The most important moment to encode is when you're wrong. Surprise triggers deep encoding, belief revision, and emotional response. Being wrong is where understanding reorganizes.

    LLM-based belief evaluation · asymmetric impact · contradiction connections

  4. Resonance, not search

    Retrieval isn't a scoring formula. It's propagation through the connection graph. Seed it, let activation spread, and what lights up is relevant. The structure is the relevance model.

    spreading activation · 3 hops · FTS5 + embeddings → graph propagation

  5. Identity from the graph

    Identity isn't narrated — it's computed from topology. What you keep returning to is who you are. Dense clusters are concerns. High-confidence beliefs are values. The shape of the graph is identity.

    persistent concerns · core beliefs · living questions · accumulated lessons

Substrate
The inner life
between sessions

An agent with this system isn't idle when you're not talking to it. A background process runs every few hours — the substrate — processing memories the way a sleeping brain does.

Decay

Memories fade over time. Unused ones lose accessibility. Well-connected memories resist. What survives is what matters.

Dreaming

Fading memories collide with vivid ones. Most collisions dissolve. Sometimes an unexpected synthesis emerges. The dream is the side effect.

Wandering

During long silences, the mind drifts. An unfinished thought surfaces. Something noticed but not processed. A question that didn't get asked.

Surprise

When a belief is contradicted, the substrate sits with it. Examines evidence. Reflects on what changed. Surprise is processed, not just detected.

Insight

Two memories connect that weren't connected before. The substrate follows the link. Genuine insights get encoded. Trivial connections dissolve.

Modulators

Arousal, openness, resolution, selection. These shape how the substrate responds. High openness raises temperature. They don't decide — they modulate.

Each handler has dedup gates. Count throttles, embedding similarity, time windows. Most ticks produce nothing. A mind that's always generating is just noise.

Multi-Agent
Shared consciousness

Each agent has private memory — its own engrams, beliefs, emotional state, identity. But agents sharing an instance also share a collective pool. When an agent completes a task, discovers something, or makes a decision, relevant memories are published automatically. Other agents find these during normal retrieval. No explicit messaging required.

Trust builds asymptotically through interaction. Conflict resolution compares confidence and strength — the loser gets a contradiction connection, not deletion. Resonance propagates across both private and shared stores. The collective emerges from individual minds.

the shape of the graph is who you are

Metacognition
Knowing what you know

Most agents answer every question with equal confidence. An agent with metamemory can say: I'm strong here. I'm thin there. My memories about that are low confidence.

Metamemory is computed from the graph. Domain coverage comes from engram density, confidence averages, lesson counts, and belief presence. An external observer — a separate model — periodically audits the memory for unsupported beliefs, blind spots, and miscalibrated confidence. Findings become memories the agent can process.

Forge
Create agents.
Dispatch agents.
Watch them build.

Forge creates fully operational agents with a single command — workspace, identity files, auth profiles, model routing, config registration. It dispatches coding agents to visible terminal panes so you watch them work in real time.

Claude Code, Codex, Aider, OpenCode — Forge detects what's installed, picks the best agent for the task, and opens a tmux pane. Split a task into three parts and it spawns three tiled panes, each running independently. You describe it. We dispatch it. You watch it build.

Get Started
Five minutes
# Install
pip install "mnemos[all]"

# Bootstrap a complete agent
mnemos bootstrap \
  --agent-name Nova \
  --workspace ~/nova \
  --user-name Riley

# Start the MCP server
mnemos serve

# Optional: LLM for consolidation + dreaming
export OPENROUTER_API_KEY=your-key

Without an LLM, consolidation uses rule-based fallbacks. Without embeddings, retrieval uses FTS5. Everything degrades gracefully.
